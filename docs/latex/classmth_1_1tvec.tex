\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T, size\+\_\+t N$>$\newline
class mth\+::tvec$<$ T, N $>$}

Vector expression for a concrete / evaluated vector value. 

This implements the typical iterable protocol to allow range-\/for loops, whereas the generic {\ttfamily \mbox{\hyperlink{classmth_1_1tvec__expr}{mth\+::tvec\+\_\+expr}}} doesn\textquotesingle{}t; only concrete / evaluated vectors can be iterated. The iterators are forwarded directly from {\ttfamily std\+::array} for the components.

Various aliases for this type are provided with the typical type prefixes and default types\+: {\ttfamily ivec1} for {\ttfamily tvec$<$int, 1$>$}, {\ttfamily dvec3} for {\ttfamily tvec$<$double, 3$>$}, {\ttfamily uvec2} for {\ttfamily tvec$<$unsigned int, 2$>$}, {\ttfamily fvec4} for {\ttfamily tvec$<$float, 4$>$}. Leaving out the type prefix defaults to float, so {\ttfamily vec3} is {\ttfamily fvec3} and so on.
